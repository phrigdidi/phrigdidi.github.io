what is the difference between non-semantic and semantic tags in html?

non-semantic tags like <div> and <span> are generic containers that tell the browser nothing about their content. semantic tags, however, clearly describe their purpose to both the browser and developer. while the browser renders both types of tags the same way, semantic tags improve code readability, accessibility, and seo. for example, a screen reader can identify a <nav> element as a navigation section, but a <div> provides no such context. semantic tags make your html more meaningful and easier to maintain.

why should we use semantic tags?

semantic tags make your html more intuitive and self-documenting, which helps developers understand the structure of your page at a glance. they also improve accessibility by providing clear landmarks for screen readers and assistive technologies. additionally, semantic html helps search engines better understand your content, which can boost your site's visibility and ranking.
